    deploy:

  branch: #发布的分支，默认是 master

    先进行安装npm install hexo-deployer-sftp --save

    deploy:
  type: sftp
  host: <host> # 远程主机的地址
  user: <user> # 用户名
  pass: <password> # 密码
  remotePath: [remote path] # 发布到的路径
  port: [port] # 端口 默认为22

经过如上配置后，执行hexo -d后就会将生产的静态文件发布到git和服务器上。

草稿

有时候建立一些草稿，还没写完不需要发布的时候我们可以使用以下的方式来实现。

1
2

# 新建草稿
hexo new draft 'md_name'

执行这个命令后会在source/_drafts文件夹下新建一个md_name.md文件，所以我们不需要加文件后缀。

然后我们启动hexo后并不能看到草稿，如果需要看到草稿可以使用以下方案

  • 修改配置文件

    将配置文件中的render_drafts改为true

  • 启动hexo时加参数

    1

    hexo server --drafts

当我们写完草稿后将其变成正式文章，执行以下命令：

1

hexo publish 'md_name'

这里也不需要加后缀名，如果文件名称是中文，在shell中无法输入中文可以执行以下操作

先对~/.bash_profile配置文件进行修改，添加或修改这一行信息：

1

export LANG=en_US.UTF-8

然后执行source ~/.bash_profile，让配置生效。

最后重新打开终端就可以输入中文了。
